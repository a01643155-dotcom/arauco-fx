INSTALACIÓN CON ANACONDA + VS CODE
────────────────────────────────────

PASO 1 – Abrir Anaconda Navigator
   • Busca "Anaconda Navigator" en tu menú de inicio
   • Abre el programa

PASO 2 – Crear entorno virtual nuevo (recomendado)
   • En Anaconda Navigator → "Environments" → "Create"
   • Nombre: arauco_env
   • Python: 3.11
   • Clic en "Create"

PASO 3 – Activar entorno e instalar Flask
   Opción A (Terminal Anaconda):
      conda activate arauco_env
      pip install flask

   Opción B (Anaconda Navigator):
      • Selecciona "arauco_env" en Environments
      • Clic en "Open Terminal"
      • Ejecuta: pip install flask

PASO 4 – Abrir el proyecto en VS Code
   • Abre VS Code
   • File → Open Folder → selecciona la carpeta "arauco_project"
   • VS Code debería detectar el entorno Anaconda automáticamente
   • Si no: Ctrl+Shift+P → "Python: Select Interpreter" → arauco_env

PASO 5 – Correr el servidor
   Opción A (Terminal de VS Code):
      • View → Terminal (Ctrl+`)
      • python app.py

   Opción B (click derecho en app.py):
      • Right-click → "Run Python File in Terminal"

PASO 6 – Abrir en el navegador
   • Abre Google Chrome o Firefox
   • Ve a: http://127.0.0.1:5000
   • ¡Listo! La aplicación ya está corriendo.

SOLUCIÓN DE PROBLEMAS
──────────────────────
❌ Error: "ModuleNotFoundError: No module named 'flask'"
   → Solución: pip install flask

❌ Error: "Address already in use / Port 5000"
   → Solución: Cambia el puerto en app.py
     app.run(debug=True, port=5001)
   → Luego ve a http://127.0.0.1:5001

❌ Las gráficas no aparecen / Los estilos no cargan
   → Verifica que estás corriendo desde DENTRO de la carpeta arauco_project
   → Verifica que las carpetas static/css/ y static/js/ existen con sus archivos

❌ macOS Error: "Port 5000 in use"
   → En macOS, el puerto 5000 lo usa AirPlay. Cambia a port=5050
